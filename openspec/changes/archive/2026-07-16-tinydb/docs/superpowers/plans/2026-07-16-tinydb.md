# tinydb 实现计划

## 概述

本计划基于设计文档和任务清单，将 tinydb（嵌入式关系型数据库）的实现分解为可执行的步骤。按依赖关系组织，每个任务包含目标、文件路径、实现要点和验证方式。

**实现顺序原则**：底层基础设施 → 核心引擎 → 上层功能 → 集成测试

**验证方式**：
- 插入记录后能正确读取

### 任务 3.3 — 实现 BufferPool

**目标**：提供页缓存，减少磁盘 I/O，支持 LRU 淘汰和脏页回写。

**文件路径**：
- `src/tinydb/storage/buffer_pool.py`

**实现要点**：
- 默认容量：100 页
- LRU 淘汰：使用 `OrderedDict` 或自定义双向链表 + 哈希表
- 方法：
  - `get_page(page_id) -> Page`（缓存命中直接返回，未命中从 FileManager 加载）
  - `mark_dirty(page_id)`
  - `flush_page(page_id)`（写回磁盘）
  - `flush_all()`（全部脏页写回）
  - `evict()`（淘汰最久未使用且非脏页，脏页先写回）
- 淘汰策略：脏页优先写回而非直接丢弃

**验证方式**：
- 重复读取同一页不触发磁盘 I/O（可 mock FileManager 验证）
- 超出容量时正确淘汰
- 脏页淘汰时自动写回
- `test_buffer_pool.py` 通过

### 任务 5.3 — 实现 Parser（语法分析器）

**目标**：递归下降解析器，将 Token 流转换为 AST。

**文件路径**：
- `src/tinydb/parser.py`

**实现要点**：
- 文法规则（按优先级）：
  ```
  statement   := ddl | dml | txn_cmd
  ddl         := CREATE TABLE ... | DROP TABLE ... | CREATE INDEX ...
  dml         := INSERT ... | SELECT ... | UPDATE ... | DELETE ...
  txn_cmd     := BEGIN | COMMIT | ROLLBACK
  condition   := expr [(AND | OR) expr]*
  expr        := term comp_op term
  ```
- 递归下降方法：`parse_statement()` → `parse_ddl()` / `parse_dml()` / `parse_txn()`
- SELECT 解析：列列表、FROM、可选 WHERE/ORDER BY/LIMIT/OFFSET
- WHERE 条件：支持 AND/OR 嵌套，比较运算符
- 错误恢复：遇到意外 token 抛出 `ParseError` 并标注位置

**验证方式**：
- `SELECT * FROM t WHERE a = 1` 生成正确 AST
- `INSERT INTO t VALUES (1, 'x')` 生成正确 AST
- 语法错误抛出 `ParseError`
- `test_parser.py` 通过

## 阶段 7：B-tree 索引

### 任务 7.1 — 实现 B-tree 节点结构

**目标**：定义 B-tree 节点的内存和磁盘表示。

**文件路径**：
- `src/tinydb/index/btree.py`

**实现要点**：
- 节点 = 一个 Page（4096 bytes）
- Header（16 bytes）：`is_leaf`(1), `key_count`(2), `page_id`(4), `parent_id`(4), `right_ptr`(4), `padding`(1)
- 内部节点：`[key, child_page_id, ...]`
- 叶子节点：`[key, record_pointer, ...]`
- 阶数约 250（INT 键 + pointer = 8 bytes/entry）
- 方法：
  - `is_full() -> bool`
  - `insert_key(key, pointer)`
  - `search(key) -> pointer`

**验证方式**：
- 节点序列化/反序列化正确
- `is_full()` 在键数达到阶数时返回 True
- `test_btree.py` 通过

---

### 任务 7.2 — 实现 B-tree 插入与分裂

**目标**：支持向 B-tree 插入键值对，节点满时自动分裂。

**文件路径**：
- `src/tinydb/index/btree.py`（追加）

**实现要点**：
- `BTree` 类：
  - `insert(key, record_pointer)`
  - `_insert_nonfull(node, key, pointer)`
  - `_split_child(parent, index)`
  - `_split_leaf(node)` / `_split_internal(node)`
- 分裂流程：
  1. 创建新节点
  2. 中位数 key 提升到父节点
  3. 原节点保留前半，新节点获得后半
  4. 叶子层：新节点加入链表（更新 right_ptr）
- 根节点分裂：树高加 1

**验证方式**：
- 插入后能查到
- 大量插入后树保持平衡
- 分裂后父节点包含正确的中位数 key
- `test_btree.py` 通过

### 任务 7.3 — 实现 B-tree 点查询与范围扫描

**目标**：支持等值查找和范围查询。

**文件路径**：
- `src/tinydb/index/btree.py`（追加）

**实现要点**：
- `search(key) -> record_pointer`：从根递归向下
- `range_scan(low, high) -> List[record_pointer]`：
  1. 找到 low 所在的叶子节点
  2. 沿 right_ptr 扫描直到 key > high
- 时间复杂度：点查询 O(log n)，范围扫描 O(log n + k)

**验证方式**：
- 插入后 `search` 返回正确 pointer
- 范围扫描返回 [low, high] 内的所有记录
- `test_btree.py` 通过

## 阶段 8：事务管理器

### 任务 8.1 — 实现 WAL 日志格式与写入

**目标**：实现 Write-Ahead Log 的格式定义和追加写入。

**文件路径**：
- `src/tinydb/transaction/wal.py`

**实现要点**：
- WAL 文件：`{db_path}.wal`
- WAL Header：`magic`(4, "WAL\0"), `version`(2), `page_size`(2), `checkpoint`(4)
- Log Record：`txn_id`(4), `op_type`(1), `page_id`(4), `data_len`(2), `old_data`(data_len), `new_data`(data_len), `checksum`(4)
- op_type 枚举：BEGIN(1), INSERT(2), UPDATE(3), DELETE(4), COMMIT(5), ABORT(6), CHECKPOINT(7)
- 方法：
  - `log_begin(txn_id)`
  - `log_write(txn_id, page_id, old_data, new_data)`
  - `log_commit(txn_id)`
  - `log_abort(txn_id)`
  - `flush()`：强制刷盘
- Checksum：简单 CRC32 校验

**验证方式**：
- 写入后读取 WAL 记录内容一致
- 多条记录追加写入顺序正确
- `test_wal.py` 通过

### 任务 8.2 — 实现事务控制（BEGIN/COMMIT/ROLLBACK）

**目标**：提供事务生命周期管理。

**文件路径**：
- `src/tinydb/transaction/txn.py`

**实现要点**：
- `TransactionManager` 类：
  - `begin() -> txn_id`
  - `commit(txn_id)`
  - `rollback(txn_id)`
- 事务与 BufferPool 集成：事务中修改的页标记为脏
- BEGIN → 分配 txn_id，写 WAL BEGIN 记录
- COMMIT → 写 WAL COMMIT 记录，刷 WAL 到磁盘，标记事务完成
- ROLLBACK → 按 WAL 逆序用 old_data 恢复页，写 ABORT 记录
- 事务 ID 自增分配

**验证方式**：
- BEGIN 后 COMMIT，修改持久化
- BEGIN 后 ROLLBACK，修改被撤销
- 未提交事务的修改对其他操作不可见（单线程简化）
- `test_transaction.py` 通过

### 任务 8.3 — 实现崩溃恢复

**目标**：启动时通过 WAL 恢复到一致状态。

**文件路径**：
- `src/tinydb/transaction/wal.py`（追加 `recover` 方法）

**实现要点**：
- `WalReader` 类：
  - `parse_records() -> List[LogRecord]`
- 恢复流程：
  1. 找到最后一个 CHECKPOINT 位置
  2. 从 checkpoint 开始重放 WAL（应用已提交事务）
  3. 丢弃未提交事务的记录
  4. 将已提交事务的 new_data 写回对应页
- 在 Database 初始化时自动调用恢复

**验证方式**：
- 模拟崩溃（不调用 close）后重启，已提交数据保留
- 未提交事务的数据被回滚
- `test_transaction.py` 中恢复测试通过

### 任务 8.4 — 实现 Checkpoint

**目标**：定期将 WAL 中的已提交修改刷回主数据库文件。

**文件路径**：
- `src/tinydb/transaction/wal.py`（追加 checkpoint 逻辑）

**实现要点**：
- 每 100 个事务自动触发
- Checkpoint 流程：
  1. 暂停新事务开始
  2. 将 WAL 中已提交事务的修改刷回主 `.db` 文件
  3. 清空 WAL 文件（截断）
  4. 更新 WAL Header 中的 checkpoint 指针
  5. 恢复接受新事务
- Checkpoint 时写 CHECKPOINT 记录到 WAL

**验证方式**：
- 触发 checkpoint 后 WAL 被清空
- checkpoint 后数据仍完整
- `test_transaction.py` 中 checkpoint 测试通过
